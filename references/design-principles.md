# Design Principles

## Must Do

- Give each slide exactly one core point.
- Make titles clear, strong, and quickly understandable.
- Build obvious visual hierarchy: title, key message, support, details.
- Use large enough text for presentation viewing.
- Keep generous whitespace and safe margins.
- Balance text and visuals; do not fill every slide edge to edge.
- Keep palette, components, icons, shadows, radii, and spacing consistent.
- Make preview and export visually consistent.
- Preserve editability: users must be able to change text and replace images.

## Avoid

- Tiny text, dense paragraphs, and busy slides.
- Full-slide AI-generated images used as the actual PPT source.
- Cheap tech blue, overused blue-purple neon, excessive glow lines, generic particles.
- The same card pile layout on every slide.
- Template-website composition.
- Fake-looking AI people when realistic editorial or product visuals would work better.
- Visuals that are difficult to recreate in HTML/CSS.
- Sacrificing editability for one-time prettiness.

## Practical Heuristics

- One strong layout idea beats many decorative effects.
- Use fewer colors with clearer roles.
- Let diagrams and data breathe; do not wrap every item in a card.
- Prefer real subject cues over abstract decoration.
- If a slide feels noisy, remove elements before adding new ones.
