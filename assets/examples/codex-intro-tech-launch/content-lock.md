# Content Lock

## PPT 目标

帮助第一次接触 Codex 的开发者、产品经理和技术团队负责人理解：Codex 不是聊天工具，而是能进入项目、修改文件、运行命令、验证结果的行动型 AI Agent。

## 目标受众

- 开发者
- 产品经理
- 技术团队负责人
- 想把 AI 从问答带入真实工作流的新用户

## 最终页数

10 页。

## 整体叙事

先用强观点建立认知差，再通过对比、闭环、架构、示例、适用任务、使用清单和协作心智，让用户知道如何把第一个清晰任务交给 Codex。

## 每页锁定

1. Cover：Codex 入门。核心观点：不是聊天，而是把任务交给能执行的 AI Agent。
2. Big idea：从“问 AI”到“让 AI 做”。核心观点：Codex 的变化是从回答转向执行。
3. Comparison：ChatGPT 和 Codex 的区别。核心观点：对话工具与行动型 Agent 的差异。
4. Process：Codex 的工作闭环。核心观点：理解目标、读代码、改文件、跑验证、汇报结果。
5. Architecture：它如何进入你的项目。核心观点：Codex 通过工作区、终端、文件系统和工具协作。
6. Code demo：一次典型任务长什么样。核心观点：好任务描述包含目标、约束和验收方式。
7. Data dashboard：Codex 适合处理哪些任务。核心观点：可验证、可拆解、可运行的任务最适合。
8. Checklist：写好任务的 5 个要点。核心观点：给目标、上下文、约束、验收并允许迭代。
9. Quote：把 Codex 当成同事，而不是搜索框。核心观点：最佳使用方式是协作、检查和迭代。
10. Closing：把第一个任务交给 Codex。核心观点：从一个小而清晰、能验证的任务开始。

## 不可随意改动

- “从聊天到执行”的核心叙事。
- Codex 是行动型 Agent 的定位。
- HTML/CSS/JS 可编辑源文件优先。
- 新野兽派视觉方向。

## 可以优化表达

- 每页标题措辞。
- 示例 prompt 的具体内容。
- 图表/流程图的局部标签。
