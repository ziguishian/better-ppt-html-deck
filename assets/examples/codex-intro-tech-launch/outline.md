# Codex 入门：从聊天到让 AI 真正干活

## Deck Brief

- 用途：Codex 入门介绍 / 产品能力演示
- 目标受众：希望理解 Codex 工作方式的开发者、产品经理、技术团队负责人
- 页数：10 页
- 比例：16:9
- 语言：中文
- 场景：科技发布会风格的演示型 PPT
- 内容密度：中低密度，强调一句话观点和强视觉中心

## Narrative

这套 PPT 先建立认知差：Codex 不是“更会聊天的助手”，而是能进入工作区、理解代码、修改文件、运行命令、验证结果的行动型 Agent。随后用工作流、能力边界、真实使用方式和安全协作机制，帮助新用户形成可操作的使用心智。

## Slides

### 1. Cover

- 标题：Codex 入门
- 核心观点：不是聊天，而是把任务交给能执行的 AI Agent。
- 页面类型：cover
- 信息层级：标题 > 副标题 > 发布会式关键词
- 视觉建议：深色发布会背景，中央强标题，局部光束或界面碎片作为主视觉。

### 2. Big Idea

- 标题：从“问 AI”到“让 AI 做”
- 核心观点：Codex 的关键变化是从回答问题变成处理任务。
- 页面类型：big-idea
- 信息层级：一句大观点 + 三个短标签
- 视觉建议：左右对比的动势线，从 chat bubble 转向 workspace。

### 3. Comparison

- 标题：ChatGPT 和 Codex 的区别
- 核心观点：ChatGPT 偏对话，Codex 偏执行。
- 页面类型：comparison
- 信息层级：两列对比：输入方式、工作对象、输出结果、验证方式
- 视觉建议：左侧克制，右侧更亮，突出 Codex 的行动闭环。

### 4. Process

- 标题：Codex 的工作闭环
- 核心观点：理解目标、读代码、改文件、跑验证、汇报结果。
- 页面类型：process
- 信息层级：5 个步骤
- 视觉建议：代码渲染流程图，节点少而清晰。

### 5. Architecture

- 标题：它如何进入你的项目
- 核心观点：Codex 通过工作区、终端、文件系统和工具调用协作。
- 页面类型：architecture
- 信息层级：用户目标、工作区、工具、验证、交付
- 视觉建议：架构图用 SVG/HTML 生成，不使用整页图片。

### 6. Code Demo

- 标题：一次典型任务长什么样
- 核心观点：好任务描述包括目标、约束、验收方式。
- 页面类型：code-demo
- 信息层级：示例 prompt + Codex 行动清单
- 视觉建议：终端窗口和代码块并排。

### 7. Data Dashboard

- 标题：Codex 适合处理哪些任务
- 核心观点：越是可验证、可拆解、可运行的任务，越适合 Codex。
- 页面类型：data-dashboard
- 信息层级：4 类任务卡片：修 bug、写功能、改文档、跑验证
- 视觉建议：四个大卡片，图标统一，避免堆文字。

### 8. Checklist

- 标题：写好任务的 5 个要点
- 核心观点：给目标、给上下文、给约束、给验收、允许迭代。
- 页面类型：checklist
- 信息层级：5 条检查项
- 视觉建议：大号编号列表，发布会式留白。

### 9. Quote

- 标题：把 Codex 当成同事，而不是搜索框
- 核心观点：最好的使用方式是协作、检查和迭代。
- 页面类型：quote
- 信息层级：一句金句 + 一行解释
- 视觉建议：极简深色页，强排版，减少装饰。

### 10. Closing

- 标题：现在，把第一个任务交给 Codex
- 核心观点：从一个小而清晰、能验证的任务开始。
- 页面类型：closing
- 信息层级：行动号召 + 三个起步建议
- 视觉建议：回到封面视觉语言，形成闭环。
